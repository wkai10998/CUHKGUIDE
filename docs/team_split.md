# 团队分工与交付清单（4位同学）

## 目标
围绕三大主界面完善内容质量：
1. 网申地图（首页阶段卡片）
2. 操作步骤（各阶段步骤细节）
3. 交流区（QA与评论）

## 分工建议

### 同学A：阶段内容负责人（prep/docs）
- 负责 `prep`、`docs` 两个阶段的步骤内容细化
- 每个步骤需补全：教程3-5条、注意事项2-3条、材料清单2-4条
- 交付：Markdown 文档 + 可直接粘贴的结构化 JSON

### 同学B：阶段内容负责人（apply/fill）
- 负责 `apply`、`fill` 阶段详细步骤
- 补充常见错误和规避建议（每步至少1条）
- 交付：Markdown 文档 + 结构化 JSON

### 同学C：阶段内容负责人（submit/wait）
- 负责 `submit`、`wait` 阶段
- 重点补：截止时间提醒、状态跟进策略、补件应对
- 交付：Markdown 文档 + 结构化 JSON

### 同学D：交流区内容运营负责人
- 补充首批 Q&A（至少 25 条）
- 分类建议：材料、语言、申请流程、时间线、提交后、结果阶段
- 每条包含：标题、分类、标准答案（2-4句）
- 交付：CSV 或 JSON

## 统一交付格式
- 文件编码：UTF-8
- 建议文件：`docs/content_payload_姓名.json`
- 字段命名统一用小写下划线

## 验收标准
1. 信息来源清晰（尽量带来源备注）
2. 文案可直接给申请者阅读
3. 不出现明显互相矛盾的数据
4. 每位同学至少交付 1 份结构化文件
